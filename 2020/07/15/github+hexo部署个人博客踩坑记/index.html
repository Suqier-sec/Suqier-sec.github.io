1. git

  1. 安装

    • yum update && yum install git -y

    • 安装完成后进行验证

    •

    • 如果这里使用的是阿里云的服务器的话是自带有git的，完全不用安装。

  2. 配置

    git config --global user.name "name"	# 这里的name填的是githhub的用户名
git config --global user.email "emali" # emalie则是github的邮件
git config user.name	# 输出绑定的用户
git config user.email	# 输出绑定的邮箱
# 生成免密key
ssh-keygen -t rsa -C "emali"	
# emali 指的是上面的github邮件，这一步主要是生成ssh密钥，后续将博客同步到github时用的

    • 验证

    •

    • 最后面的生成ssh密钥已经生成过了，就不再次生成了。

2. 安装node.js

yum update gcc -y	# 安装gcc后续用来编译安装node.js
yum install gcc+ gcc-c++ -y
wget https://npm.taobao.org/mirrors/node/v10.13.0/node-v10.13.0.tar.gz	# 下载node.js的安装文件
tar xvf node-v10.13.0.tar.gz
cd node-v10.13.0/
./configure	# 生成文件
make	# 编译这一步超级久，看服务器配置吧，反正我编译了一个钟
make install# 安装
  • 上面的操作都已经装过了,且不方便再重复一次,这里不贴图了。

3. 安装hexo

cd /root && mkdir hexo	# 切换到root目录下生成hexo文件
npm config set registry https://registry.npm.taobao.org	# 将npm源更换为淘宝的,如果不更换,默认源下载慢到怀疑人生.
cd hexo
npm install -g hexo-cli #安装hexo
hexo init myblog # hexo 生成myblog文件夹在其中进行初始化
npm install
hexo g	# 生成静态文件
  • 访问http://localhost:4000 就可以看到生成的博客了。
  • 设置主题
  • hexo 官网上提供了三百多款主题，想要哪个可以自己选择：https://hexo.io/themes/ (没有科学上网的话访问较慢)
git clone 主题链接
tar -xvf 主题压缩包
# 解压之后会有一个主题名
vim /root/hexo/myblog/_config.yml
在theme：后换上解压出来的主题名就算是应用主题了。
hexo clean	# 清除静态文件
hexo g	# 生成新主题的静态文件
  •

4. 安装宝塔+Nginx

  • 安装宝塔
yum install -y wget && wget -O install.sh http://download.bt.cn/install/install_6.0.sh && sh install.sh
  •

  • 安装配置Nginx

  • 在软件商店搜索nginx，那个免费的就是了。因为安装过了，显示的是卸载。

  •

  • 下载之后点击右侧的设置，修改配置。

  • 第一行改成 user root www 这里的用户不是root的话可能没有权限

  •

  • 第六十五行改成也就是nginx网站根目录root /root/hexo/myblog/public;

  •

  • 再返回nginx管理，将nginx进行重启。

  •

  • 到这一步就已经配置好网站了，可以使用ip或域名的形式访问前面部署好的hexo了。

5. 部署到github

  • 新建仓库，访问github.com 在首页点击创建仓库。（！！！仓库名前缀一定要和用户名一致，不然会连接不上）

  •

  • 接下来就是修改hexo的配置文件，配置好github链接

  • vim /root/hexo/myblog/_config.yml

# 将光标下滑至文件的末尾
deploy:
  type: git
  repository:
    github: git@github.com:xiaoming/xiaoming.github.io.git	# 这里的xiaoming改成自己的仓库名
  branch: master
  •

  • 安装同步到github的插件npm install hexo-deployer-git --save

  • 查看前面生成的ssh密钥cat /root/.ssh/id_rsa.pub将其复制

  •

  • 返回github点击右上角的头像，然后settting，再ssh key and gpg keys 添加新的ssh密钥

  •

  • 返回服务器，在终端输入 ssh -T git@github.com 验证连接。提示这个显示连接成功。

  •

6. 发布文章

cd /root/hexo/myblog	# 切换到hexo环境下
hexo new hello-world	# 生成名为hello-world的文章
hexo clean	# 清除静态文件
hexo g	# 生成静态文件
hexo d	# 同步到github
  • 几分钟后访问xiaoming.github.io就可以看到自己的发布的文章了。(这里的xiaoming为自己的用户名)

7. 踩坑总结

  • 默认的是next的主题，很简洁，如果要个性化的话，可以去hexo访问的主题商店，很多主题都会有文档的，有的写的很详细，有的一笔带过。这里不推荐第一次接触就去弄那种很复杂的主题，因为会怀疑人生了。(我就是折腾了大几天，自闭了都)
  • 还有发布文章要内嵌图片的话，需要用markdown格式的链接。有个大坑需要注意，白嫖gitee的仓库做图床的话，访问着没有登录gitee账号是无法加载图片的。本博客用的是阿里云oss+picgo做的图床。(阿里云oss如何部署图床自行百度，这里不写出来了。)
  • hexo常用命令缩写
hexo g # 完整命令为hexo generate,用于生成静态文件
hexo s # 完整命令为hexo server,用于启动服务器，主要用来本地预览
hexo d # 完整命令为hexo deploy,用于将本地文件发布到github上
hexo n # 完整命令为hexo new,用于新建一篇文章
------ 分割线 -------
hexo new “postName” # 新建文章
hexo new page “pageName” # 新建页面
hexo generate # 生成静态页面至public目录
hexo server # 开启预览访问端口（默认端口4000，“ctrl + c”关闭server）
hexo deploy # 部署到GitHub
hexo help # 查看帮助
hexo version # 查看Hexo的版本
hexo s -g # 生成并本地预览
hexo d -g # 生成并上传
