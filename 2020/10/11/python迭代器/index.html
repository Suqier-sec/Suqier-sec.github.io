python迭代对象与迭代器

python迭代对象与迭代器

迭代对象

  • 可迭代对象：

    • 字面意思：对象？python中一切皆对象，一个实实在在的值，对象。
    • 可迭代：更新迭代，重复的，循环的一个过程，更新迭代每次都有新的内容。
    • 可以进行循环更新的一个实实在在的值。
    • 专业角度：可迭代对象？内部含有__iter__方法的对象，就为可迭代对象。
    • 以接触的可迭代对象：str/list/tuple/dict/set/range/文件句柄。

  • 获取一个对象的所有方法，并且以字符串的形式返回：dir()。

  • 判断一个对象是否为可迭代对象:__iter__。

    # 获取一个对象的所有方法: dir()

# s1 = 'aserq51234'
# print(dir(s1))
l1 = [1, 2, 3]
# print(dir(l1))
# print("__iter__" in dir(l1))
print("__iter__" in dir(range(10)))

  • 小结

    • 优点：
      • 存储的数据比较直观。
      • 拥有的方法比较多，操作方便。
    • 缺点：
      • 占用内存。
      • 不能通过for循环，不能直接取值(索引，key)。

迭代器

  • 迭代器的定义

    • 字面意思：更新迭代，器：工具。更新迭代的工具。
    • 专业角度：内部含有__iter__方法且含有__next__方法的对象就是迭代器。
    • 判断是否为迭代器：__iter__ and __next__ in dir(obj)
    f = open("test.txt",'w+')
print("__iter__" in dir(f) and "__next__" in dir(f))
f.close()

  • 可迭代对象如何转换成迭代器

    • obj.__iter__ or iter(obj)
    
# 可迭代对象可以转化成迭代器

s1 = '123as'
obj = s1.__iter__()
print(obj.__next__())  # 每迭代一次只取一个值
print(obj.__next__())
print(obj.__next__())
print(obj.__next__())
print(obj.__next__())

l1 = [11, 22, 33, 44, 55, 66]
res = l1.__iter__()

for i in range(len(l1)):
    print(res.__next__())

  • while 循环模拟for循环机制

    # 利用while循环模拟for循环对可迭代对象取值的机制     ** 面试常考
l1 = [11, 22, 33, 44, 55, 66, 77, 88, 99, 12, 13, 14]
obj = iter(l1)  # 将可迭代对象转换成迭代器
while True:
    try:
        print(next(obj))    # 利用next取值
    except StopIteration:   # 捕抓超出范围异常
        break

  • 小结

    • 优点
      • 节省内存。
      • 惰性机型：next一次取一次值，绝不多取。
    • 缺点：
      • 不能直观的查看里面的数据，速度慢。
      • 取值不走回头路，只能一直向下取值。

  • 可迭代对象和可迭代器的对比：

    • 可迭代对象是一个操作方法比较多，较为直观存储数据相对小（几百万数据以下，8G内存是足够的）的数据集。
    • 当侧重对于数据可以灵活处理，并且内存空间足够多，将数据集设置为可迭代对象是明确的选择
    • 迭代器是一个非常节省内存，可以记录取值位置，可以通过循环+next方法取值，但是不直观，操作方法比较打单一的数据集
    • 当数据量过大，大到可撑爆内存，或者以节省内存为首选因素时，将数据集设置位迭代器是一个不错的选择
